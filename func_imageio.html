Image input/output operations

A set of functions to read, write, and display images represented as NumPy arrays.

Convert image

Parameters

machinevisiontoolbox.base.imageio.float_image(image, floatclass='float32')[source]

Convert image to float type

Parameters

  • image (ndarray(h,w,nc) or ndarray(h,w,nc)) – input image

  • floatclass (str) – ‘single’, ‘double’, ‘float32’ [default], ‘float64’

Returns

image with floating point pixel types

Return type

ndarray(h,w,nc) or ndarray(h,w,nc)

  • float_image() is a copy of image with pixels converted to

    float32 floating point values spanning the range 0 to 1. The input integer pixels are assumed to span the range 0 to the maximum value of their integer class.

  • float_image(im, floatclass) as above but with floating-point pixel

    values belonging to the class floatclass.

Example:

>>> from machinevisiontoolbox import iread, idisp, float_image
>>> im, file = iread('flowers1.png')
>>> idisp(float_image(im))
<AxesSubplot:title={'center':'Machine Vision Toolbox for Python'}>

Note

  • Works for greyscale or color (arbitrary number of planes) image

  • If the input image is integer the pixel values are scaled from an input range spanning zero to the maximum positive value of the output integer class to [0,1]

  • If the input image is a floating class then the pixels are cast

    to change type but not their value.

References
  • Robotics, Vision & Control, Section 12.1, P. Corke,

    Springer 2011.

Example:

>>> from machinevisiontoolbox import iread, idisp
>>> im, file = iread('monalisa.png')
>>> idisp(im)
<AxesSubplot:title={'center':'Machine Vision Toolbox for Python'}>

Note

  • Greyscale images are displayed in indexed mode: the image pixel value is mapped through the color map to determine the display pixel value.

  • For grey scale images the minimum and maximum image values are mapped to the first and last element of the color map, which by default (‘greyscale’) is the range black to white. To set your own scaling between displayed grey level and pixel value use the ‘cscale’ option.

  • The title of the figure window by default is the name of the variable passed in as the image, this can’t work if the first argument is an expression.

References

machinevisiontoolbox.base.imageio.int_image(image, intclass='uint8')[source]

Convert image to integer type

Parameters

  • image (ndarray(h,w,nc) or ndarray(h,w,nc)) – input image

  • intclass (str) – either ‘uint8’, or any integer class supported by np

Returns

image with integer pixel types

Return type

ndarray(h,w,nc) or ndarray(h,w,nc)

  • int_image(image) is a copy of image with pixels converted to unsigned

    8-bit integer (uint8) elements in the range 0 to 255.

  • int_image(intclass) as above but the output pixels are converted to

    the integer class intclass.

Example:

>>> from machinevisiontoolbox import iread, idisp, int_image
>>> im, file = iread('flowers1.png')
>>> idisp(int_image(im, 'uint16'))
<AxesSubplot:title={'center':'Machine Vision Toolbox for Python'}>

Note

  • Works for greyscale or color (arbitrary number of planes) image

  • If the input image is floating point (single or double) the

    pixel values are scaled from an input range of [0,1] to a range spanning zero to the maximum positive value of the output integer class.

  • If the input image is an integer class then the pixels are cast

    to change type but not their value.

References
  • Robotics, Vision & Control, Section 12.1, P. Corke,

    Springer 2011.

machinevisiontoolbox.base.imageio.iread(filename, *args, verbose=True, **kwargs)[source]

Read image from file

Parameters

  • file (string) – file name or URL

  • kwargs – key word arguments

Returns

image and filename

Return type

tuple or list of tuples, tuple is (image, filename) where image is a 2D, 3D or 4D NumPy array

  • image, path = iread(file) reads the specified image file and returns the image as a NumPy matrix, as well as the absolute path name. The image can by greyscale or color in any of the wide range of formats supported by the OpenCV imread function.

  • image, url = iread(url) as above but reads the image from the given URL.

If file is a list or contains a wildcard, the result will be a list of (image, path) tuples. They will be sorted by path.

  • iread(filename, dtype="uint8", grey=None, greymethod=601, reduce=1, gamma=None, roi=None)

Extra options include:

  • ‘uint8’ return an image with 8-bit unsigned integer pixels in the range 0 to 255

  • ‘double’ return an image with double precision floating point pixels in the range 0 to 1.

  • ‘grey’ convert image to greyscale, if it’s color, using ITU rec 601

  • ‘grey_709’ convert image to greyscale, if it’s color, using ITU rec 709

  • ‘gamma’,G apply this gamma correction, either numeric or ‘sRGB’

  • ‘reduce’,R decimate image by R in both dimensions

  • ‘roi’,R apply the region of interest R to each image, where R=[umin umax; vmin vmax].

Parameters

  • dtype (str) – a NumPy dtype string such as “uint8”, “int16”, “float32”

  • grey (bool) – convert to grey scale

  • greymethod (int) – ITU recommendation, either 601 [default] or 709

  • reduce (int) – subsample image by this amount in u- and v-dimensions

  • gamma (float or str) – gamma decoding, either the exponent of “sRGB”

  • roi (array_like(4)) – extract region of interest [umin, umax, vmin vmax]

Example:

>>> from machinevisiontoolbox import iread, idisp
>>> im, file = iread('flowers1.png')
>>> idisp(im)
<AxesSubplot:title={'center':'Machine Vision Toolbox for Python'}>

Note

  • A greyscale image is returned as an HxW matrix

  • A color image is returned as an HxWx3 matrix

  • A greyscale image sequence is returned as an HxWxN matrix where N is the sequence length

  • A color image sequence is returned as an HxWx3xN matrix where N is the sequence length

References

  • Robotics, Vision & Control, Section 10.1, P. Corke, Springer 2011.

machinevisiontoolbox.base.imageio.iwrite(im, filename, **kwargs)[source]

Write NumPy array as image file

Parameters

  • filename (string) – filename to write to

  • kwargs – additional arguments, see ImwriteFlags

Returns

successful write

Return type

bool

  • iwrite(im, filename, **kwargs) writes im to filename with **kwargs currently for cv.imwrite() options. The file type is taken from the extension in filename.

Example:

>>> from machinevisiontoolbox import iwrite
>>> import numpy as np
>>> image = np.zeros((20,20))  # 20x20 black image
>>> iwrite(image, "black.png")
True
